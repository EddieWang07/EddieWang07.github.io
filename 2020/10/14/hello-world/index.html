豐富的分析，顯示訪客的性別、年齡層、興趣、國家等
停留了多久時間，跳離率(Bounce Rate)多少
流量在各時間的變化
知道你的網站流量從哪裡來

實作想法
目前還想不到可以直接產url 並分析網站的工具，所以還是要：
先有網站網址
使用 google analytics
埋GA
網站網址
想以最低成本，且最方便的方式有一個可供測試的網站：
Wix
wordpress
購買domain
git page
實際實作
Wix
在Wix 中就內建有 google analytics，註冊完GA 後取得追蹤id ，準備要在Wix 中埋入代碼時才發現，只有Wix 進階付費版本才有此功能。
Business Basic US$17 / 月
Wordpress
Wordpress 有內建的外掛可以直接使用GA 或是修改header footer 來放代碼，但一樣是要升級至商務版方案才能安裝外掛程式。
商務版 US$25 / 月
既然都不給免費埋GA 那我們自己改寫應該行了吧 !
購買domain
寫了一個單純可以放GA 在head 的html 接著就是放上網域了
除了付費domain 外還有免費可以供測試的 freenom，但萬事俱備後才發現… 不對啊怎麼放?
還要在短時間內研究DNS 研究如何發布，不行這要花太多時間了!!!
Git page
使用hexo 建立的git page 生成後會產生index.html
有網址、能埋ga 在中
這不就是我們要的嗎 !!
把GA 拿到的代碼貼到index.html 內後
hexo d -g
檢視頁面程式碼
不對啊GA 代碼呢 ?
回去看index.html 裡面的剛剛放的程式消失了
才想到 每次執行 hexo d -g 都會重建一份index.html
上網搜尋了一下
發現有些主題內建的 _config.yml 早已寫好GA
themes/landscape/_config.yml (landscape主題)
google_analytics: UA-xxxxxxxx-x
之後再做一次
hexo d -g
終於完成了！！！！
結語
這次是受朋友之託需要這樣的網站，實作後也練習了蠻多有趣的東西。google analytics、hexo、git page 中間跳過蠻多的，之後如果有空會再補上各個細節教學。
實現需求時總能找到更多好玩的技術，希望大家也能保持 ( 或找回 ) 當工程師的樂趣。Have fun !！
